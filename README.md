## 📝 Project Description
**SkillSwap** is a mobile-first web app that is designed for the Taylor's University student community. Our platform has a focus on the "silo" effect, whereby students are usually separated within their own faculties (e.g. a Business student might want to learn basic Coding, but has no way of meeting a Computing student).

Through a digital marketplace of skills, students can post what they can teach (i.e. Graphic Design, Calculus, or Guitar) and what they want to learn. The app facilitates "swaps," or the free exchange of knowledge through a form of peer-to-peer connection and sharing of media resources.

## ⚙️ Overview of SDLC Work

Our team followed a structured Software Development Life Cycle (SDLC) to ensure the project was delivered systematically:

### 1. Planning & Requirements
We used **Trello** as our main project management tool. We started by defining our User Stories and a Product Backlog. Following an Agile inspired method, we focused on core features such as User Authentication and Connection System to get a functioning Minimum Viable Product (MVP). Besides, we divided the development process into two phases of the sprint. The first sprint focused on creating an initial version of the system and getting user feedback while the second sprint focused on implementing improvements and improving the overall user experience based on the feedback received.

### 2. Design
There are many tools, but the one used for creating high fidelity wireframes and interactive prototypes was **Figma**, our UI/UX designer **Jun Hom**. We focused on a "Mobile-First" design philosophy, and made sure that the interface was intuitive for students that are constantly on the move around campus. The design is all about high-contrast elements and fluid transitions (the use of Framer Motion) in order to deliver a premium user experience.

### 3. Implementation
The application was built using a modern tech stack:
*   **Frontend:** React with Vite and Tailwind CSS.
*   **Backend:** Supabase for real-time database management and Authentication.
*   **Modular Architecture:** We refactored our code into a modular component structure to ensure maintainability and scalability.

### 4. Testing & Evaluation
Under **Chun Hoong’s** lead, our team performed rigorous manual testing. Our evaluation process included:
*   **User Authentication Testing:** Verifying secure login/signup flows via Supabase.
*   **Functional Testing:** Ensuring the "Swap" media upload and notification system triggered correctly across different accounts.
*   **Peer Review:** The group conducted internal code reviews and walkthroughs to identifying UI inconsistencies.
*   **Feedback Integration:** We collected informal feedback from peers to improve the "Explore" page by adding a "Rating System".
